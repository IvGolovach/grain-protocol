# Profiles

Profiles are normative constraints for v0.1:

- `cbor-profile.md` — strict DAG-CBOR + baseline limits + strict mode
- `cose-profile.md` — narrow COSE_Sign1 signature container rules
- `e2e-profile.md` — E2E envelope, deterministic nonce, manifest resolution
- `qr-profile.md` — GR1 QR pipeline
- `food-profile.md` / `food-profile.v1.json` — reducer-visible food events
- `inventory-profile.md` / `inventory-profile.v1.json` — opaque inventory events
- `audit-artifact-profile.md` / `audit-artifact-profile.v1.json` — opaque audit evidence events
- `profile-registry.v1.json` — machine-readable registry of domain profiles and samples
